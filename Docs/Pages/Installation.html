Download and Installation

Download

The official download location is https://multicommander.com/downloads. If you download it from a different source, make sure that you trust it, as you can never be sure if someone has tampered with the package.
This is especially important when downloading the portable version, as it is distributed as a simple .zip file. Never download the portable version from untrusted sources.

Installation

Full Installer (.exe)

The full installer is named MultiCommander_x64_([version]).exe or MultiCommander_win32_([version]).exe.

During the installation process, you will have two options: Single User or All Users.

If you choose Single User, the installer will, by default, select an installation path under your user account. This option does not require administrator rights.

If you choose All Users, the installer will, by default, install to the shared Program Files folder, giving access to all user accounts on the system.
This option requires administrator rights. If necessary, the installer will restart itself to request admin permission.

Even with the "All Users" option, each user will have their own configuration. Configuration files are always saved under the user’s Application Data path:
C:\Users\<username>\AppData\Roaming\MultiCommander\Config\

Hint

By typing ":goconfig" in the command line within Multi Commander, the active Explorer Panel will open the Multi Commander configuration folder.

Portable Installer (.zip)

The portable installer is named MultiCommander_x64_Portable_([version]).zip or MultiCommander_win32_Portable_([version]).zip.

The portable version is a zip archive that can be unpacked to a folder of your choice. Ensure the folder structure within the zip file is maintained.
The portable version is identical to the full version but is configured to save all settings and configurations within the folder where it’s run from, rather than saving data to the registry or user account folders.
This makes it ideal for running from a USB stick or other portable media. Just ensure that Multi Commander has write permissions to the folder from which it’s launched.

Note for WinPE Users

For using Multi Commander in WinPE 4.0 (assuming WinPE is located at X:), copy the following files to the corresponding directories before building WinPE:

C:\Windows\System32\en-EN\oledlg.dll.mui → X:\Windows\System32\en-EN\
C:\Windows\Syswow64\en-EN\oledlg.dll.mui → X:\Windows\SysWOW64\en-EN\
C:\Windows\Syswow64\oledlg.dll → X:\Windows\SysWOW64\
C:\Windows\System32\oledlg.dll → X:\Windows\System32\

Contributed by user Ulrich; tested on Windows 10 PE (x86 and x64).

Installer Parameters

The installer supports various parameters, allowing it to run silently or be controlled via scripts.

MultiCommander_x64_(14.6.0.3059).exe /S /D=[Target Folder] /PrevInstall=[Upgrade|Uninstall]
/InstallMode=[All|User] /Kill=[Ask|Auto|Abort]
  • /S - Silent
    Run the installer without displaying any UI or asking for input.
  • /D - Destination
    Specify the target installation folder. If omitted, the default location (under Program Files) is used.
    Eg. /D="D:\MyPrograms\MultiCommander"
  • /PrevInstall - Previous Installation
    Specify how to handle an existing installation: "Upgrade" or "Uninstall".
    Eg. /PrevInstall=Upgrade
  • /InstallMode - Installation Mode
    Choose to install Multi Commander for "All" users or just the current user. If installing for all users to a protected location (e.g., Program Files), ensure the installation is run as administrator.
    Eg. /InstallMode=All
  • /Kill - Handle running instances of Multi Commander during installation
    If Multi Commander is running during an upgrade, specify how to handle it:
    • /Kill=Auto - Automatically close any running instances
    • /Kill=Ask - Ask the user what to do
    • /Kill=Abort - Abort the upgrade if Multi Commander is running
    Eg. /Kill=Auto
  • /CLEAR=Uninstall - Clear Uninstall Data
    Remove all Multi Commander entries from the "Installed Programs" registry.
    Eg. /CLEAR=Uninstall
  • /FORCEDEST="Path" - Force a new installation path
    Override the path saved in the registry from a previous installation.
    Eg. /FORCEDEST="D:\NewInstallPath\"

Example

# Upgrade Current Installation
MultiCommander_x64_(14.6.0.3059).exe /S /D="C:\Bin\MultiCommander" /PrevInstall=Upgrade /InstallMode=All /Kill=Auto
